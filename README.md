# Анализ психического состояния по сообщениям пользователей

# Постановка задачи

Целью данной работы является разработка системы автоматической классификации психического состояния пользователя на основе его сообщений.  
В настоящее время задача оценки психического состояния пользователей является особенно актуальной в связи с активным использованием социальных сетей и чат-ботов. Отдельно стоит отметить рост числа сообщений о негативных психологических эффектах взаимодействия пользователей с чат-ботами (так называемый ‘ИИ-психоз’). Разработка алгоритмов автоматического анализа сообщений пользователей может помочь повысить безопасность взаимодействия человека с системами на основе генеративного ИИ.

## Формат входных и выходных данных

**Входные данные:**

* Текстовое сообщение (string). Сообщение пользователя произвольной длины. Пример: “trouble sleeping, confused mind, restless heart. All out of tune”.

**Выходные данные:**

* Метка класса (categorical label). Один из семи классов: “Normal”, “Depression”, “Suicidal”, “Anxiety”, “Stress”, “Bi-Polar”, “Personality Disorder”.

## Метрики

Выбранный датасет несбалансирован, поэтому в качестве метрик будут использоваться Balanced Accuracy (BA) и Macro F1-score. Важно распознавать одинаково хорошо все возможные состояния пользователей, поэтому метрики, учитывающие распознавание каждого класса отдельно, хорошо подходят.  
Ожидаемые значения: 

* Для бейзлайна: BA и Macro F1 около 0.6-0.7   
* Для финальной модели: BA и Macro F1 не ниже 0.8

Ожидаемые значения оценены на основе анализа решений, выложенных на Kaggle.

## Валидация и тест

Датасет несбалансирован, значит для валидации и теста необходимо сохранить исходное распределение классов. Поэтому для разделения выборок будет использоваться подход стратифицированного разделения (Stratified split). Для воспроизводимости фиксируется значение seed. Размер датасета позволяет разделить выборки train/val/test в пропорции 70/15/15. 

## Датасеты

В рамках проекта рассматривается следующий датасет: [https://www.kaggle.com/datasets/suchintikasarkar/sentiment-analysis-for-mental-health/data](https://www.kaggle.com/datasets/suchintikasarkar/sentiment-analysis-for-mental-health/data)  
Датасет включает в себя \~53000 записей (31.5 мб), каждая из которых включает в себя поля unique-id (не используется), Statement (текстовое сообщение произвольной длины; по предварительному анализу есть пропущенные и повторяющиеся значения), Mental Health Status (метка класса).  
Распределение классов:

* Normal: 31%  
* Depression: 29%  
* Suicidal: 20%  
* Anxiety: 7%  
* Stress: 5%  
* Bi-Polar: 5%  
* Personality Disorder: 2%

Основными сложностями могут стать:

* сильная несбалансированность датасета (самый популярный класс встречается в \~15 раз чаще самого непопулярного);  
* Неоднозначность меток некоторых объектов (например, сообщения класса Depression и Suicidal могут быть похожи, а в реальной жизни данные классы нередко встречаются вместе, но датасет предлагает только один результирующий класс в качестве ответа). 

# Моделирование

## Бейзлайн

В качестве бейзлайна совместно используются TF-IDF для векторизации текста и логистическая регрессия для классификации.   
Перед созданием TF-IDF представлений текст предобрабатывается. Предобработка включает следующие этапы:

1. Приведение к нижнему регистру  
2. Удаление знаков пунктуации  
3. Удаление веб\-ссылок (могут встречаться, так как часть материала – посты из социальных сетей)  
4. Токенизация  
5. Стемминг

Пример реализации аналогичного подхода: [https://www.kaggle.com/code/annastasy/mental-health-sentiment-analysis-nlp-ml\#-5.-Model-Traning-and-Evaluation-](https://www.kaggle.com/code/annastasy/mental-health-sentiment-analysis-nlp-ml#-5.-Model-Traning-and-Evaluation-) 

## Основная модель

Основной моделью является дообученная DistilBERT ([https://huggingface.co/distilbert/distilbert-base-uncased](https://huggingface.co/distilbert/distilbert-base-uncased)) с дополнительным линейным слоем для классификации. При обучении последние n слоёв (гиперпараметр, выбор определяется доступными вычислительными ресурсами; планируется использовать n=2) DistilBERT вместе с линейным слоем обучаются на задачу классификации.  
Модель DistilBERT была выбрана как компромисс между требовательными моделями и ограниченными вычислительными ресурсами. Не сильно уступая BERT, данная модель легче и быстрее, что позволяет использовать её в рамках учебного проекта.  
Для токенизации используется токенизатор DistilBERT.

# Внедрение

Модель планируется развернуть в виде REST API сервиса. Пользователь сможет отправлять текст сообщения и получать результирующий класс.  
Пример взаимодействия:

* Отправка:  
  `POST /predict`  
  `{`  
  `"text": "I feel very anxious"`  
  `}`  
* Ответ:  
  `{`  
  `"label": "Anxiety"`  
  `}`

Для реализации нужны следующие компоненты:

* HTTP-сервер (планируется использовать FastAPI для реализации)  
* Загрузка весов обученной модели  
* Валидация входных данных (например, проверка сообщения на пустоту)  
* Предобработка текста (например, токенизация)  
* Логирование  
* Мониторинг качества модели после развёртывания  
* Контейнеризация (планируется использовать Docker)
